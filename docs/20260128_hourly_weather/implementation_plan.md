# 1時間毎の天気表示の実装計画

## 1. 目的
時間別予報が縦に並んでしまっていた問題（CSSの適用ミス）を修正し、ユーザーの要望通り「横スクロールのカード形式」で美しく表示されるように改善する。

## 2. 変更内容

### 2.1. CSSの修正 (`app/api/mcp/route.ts`)
- **メディアクエリの修正**: 前回の修正で誤って `@media (prefers-color-scheme: dark)` 内に記述してしまっていた時間別表示用のスタイルを、全モード共通（ライト/ダーク両対応）の場所に移動する。
- **カードデザインの洗練**:
  - `hourly-item`: 背景色、ボーダー、パディングを調整し、よりカードらしい見た目にする。
  - `flex-shrink: 0`: 横並びを維持するためにアイテムが縮まないように設定を確実にする。
  - `box-shadow`: 軽い影を追加して浮き上がった感じを出す。

### 2.2. HTML構造の調整 (`app/api/mcp/route.ts`)
- `hourly-container` に `scroll-padding` や `padding` を追加し、スクロール時の余白を最適化する。

## 3. 検証計画

### 3.1. 手動検証
- ブラウザまたはChatGPT UI上で、ライトモードとダークモードの両方で横スクロールカードが表示されることを確認。
- 各時間別のデータが欠落なく表示されていることを確認。

## 4. リスク
- **レスポンスサイズ**: データ量が増えるため、VercelのFunction Payload制限（4.5MB）に注意が必要だが、テキストデータなので問題ない範囲と想定。
- **APIレートリミット**: Open-Meteoの制限内であることを確認（商用利用でない限り問題なし）。
